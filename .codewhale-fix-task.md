# Fix Task: Event ID daily reset

你是 CodeWhale，负责修复 Milestone A Event Layer 的审查问题。请直接修改文件，但不要 commit。

## 背景

Codex 二审结论：FAIL，需要返修。

High 问题：
`scripts/event_log.py` 的 `eventId` 不满足“daily incrementing format `EVT-YYYYMMDD-000001`”的完整要求。

当前 `.state` 只保存 `nextEventNumber`，没有保存/比较日期。到 UTC 新的一天后会继续前一天计数，例如：

```text
EVT-20260620-000005
EVT-20260621-000006  # 错误
```

正确行为应该是：

```text
EVT-20260620-000005
EVT-20260621-000001  # 正确，跨天重置
```

## 修复要求

只修改 `scripts/event_log.py`，必要时可更新 README/PROJECT_PLAN 中相关说明，但不要扩大范围。

实现要求：

1. `.state` 中保存当前日期，例如：
   ```json
   {
     "date": "20260620",
     "nextEventNumber": 6
   }
   ```
2. 分配 eventId 时使用 UTC 日期。
3. 如果 state date != 当前 UTC 日期：
   - 将计数重置为 1；
   - 生成 `EVT-当前日期-000001`；
   - 写回 state：当前日期 + nextEventNumber=2。
4. 如果 state date == 当前 UTC 日期：
   - 按 nextEventNumber 生成；
   - nextEventNumber +1 写回。
5. 兼容旧 state：如果 `.state` 里只有 `nextEventNumber` 没有 `date`，按当前 UTC 日期初始化，不要崩。
6. 继续保持跨进程锁，不能破坏并发安全。
