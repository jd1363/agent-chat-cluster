8.2 刚需增补模块（建议全部部署）
8.2.1 任务全生命周期管理
补充原因
原有架构仅支持下发需求，无标准化任务跟踪；多任务并行时，无法快速查看进度、卡点、负责人，溯源困难。
核心功能
统一任务编号、状态台账、优先级划分、任务转交、手动终止，任务与Agent、成本强绑定。
落地配置
追加全局规则（粘贴至OpenClaw规则区）

【任务管理规则】
1. 所有需求统一分配任务ID，格式：Task-三位数字（Task-001、Task-002）
2. 任务优先级：高/中/低，高优先级优先分配资源
3. 任务状态：待分配、进行中、阻塞、已完成、已终止
4. 中枢自动维护任务台账，定时公示进度；任务阻塞需标注【任务阻塞】+具体原因
专用运维命令

/task list                # 查看全量任务台账（ID、负责人、状态、成本）
/task create 优先级 任务内容  # 新建任务 示例：/task create 高 开发Python接口模块
/task status 任务ID       # 查看单任务详细进度
/task transfer 任务ID 成员名 # 任务转交
/task stop 任务ID         # 手动终止任务
8.2.2 多维度主动告警系统
补充原因
原有仅支持成本预警，Agent卡死、掉线、死循环、权限错误等故障需人工肉眼排查，发现滞后。
核心功能
分级告警、进程监控、死循环检测、弹窗提醒、独立告警日志，无需全程盯屏。
落地配置
更新中枢启动命令（整合原有参数，全平台通用）

# Mac / Linux
openclaw --web --port 8080 --enable-usage-tracking --enable-group-mode \
--enable-process-monitor --enable-deadloop-detect --enable-alert-popup \
> 全局日志/对话日志/主集群日志_$(date +%Y%m%d).log 2>&1
# Windows
openclaw --web --port 8080 --enable-usage-tracking --enable-group-mode --enable-process-monitor --enable-deadloop-detect --enable-alert-popup > 全局日志\对话日志\主集群日志_%date%.txt 2>&1
参数说明：
--enable-process-monitor：监控进程存活与响应状态
--enable-deadloop-detect：拦截Agent无限循环、无效刷屏
--enable-alert-popup：终端+Web面板双端弹窗提醒
告警日志归档

# 提前创建目录（一次性执行）
mkdir -p 全局日志/告警记录
# 开启告警日志持久化
/alert export --path "./全局日志/告警记录/每日告警.log" --append true
告警分级标准
蓝色提醒：席位空闲、任务完成、日志归档
黄色警告：预算达80%、单轮Token过高、Agent长时间闲置
红色紧急告警：进程卡死、掉线、死循环、目录权限错误、预算耗尽

补充原因
集群挂载、分组、预算、规则配置复杂，终端崩溃、误操作、重启设备后需重新部署，耗时费力。

1）纯本地逻辑（0 Token，可放心全开）
仅执行本地文件读写、进程管控、定时轮询、权限判断，不调用大模型API：
管理员审计、配置快照、批量命令、进程监控、定时任务、目录权限、席位启停、日志本地存储。
2）AI语义交互（微量Token消耗）
需要模型生成短文本内容，仅触发动作时产生消耗：
任务台账播报、文字告警提示、标签检索、/help知识库查询、任务状态标注。

9.6 核心避坑规则（防止消耗异常飙升）
禁止将完整代码、超长日志、全量历史对话全员广播，仅推送摘要/状态；
状态播报间隔最低不低于10分钟，避免高频轮询；
规则仅配置一次永久生效，不要反复粘贴长文本规则；
死循环检测仅做本地拦截+弹窗告警，不广播详细排查日志。
