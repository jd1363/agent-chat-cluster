多AI Agent集群群聊协同+成本管控+弹性扩容 全流程落地方案
文档说明
本方案在群聊可视化、AI互相对话、Token/成本计量、预算预警基础上，新增模块化弹性扩容设计，提前预留接入空间、标准化新增Agent流程、做资源隔离，支持后续持续接入新工具/新Agent，避免集群臃肿、配置混乱、权限冲突、统计错乱。
适用场景：长期迭代AI团队、按需扩充岗位、临时接入第三方编码/专用Agent，所有能力开箱即用，扩容无需重构原有架构。


一、方案整体概述
1. 核心能力清单
群聊协同：多Agent同群工作、自由对话求助、全员消息互通
全可视化：终端+Web双界面实时查看所有对话、工作产出、思考过程
成本管控：单Agent独立统计Token、计费、预算阈值预警、超支自动保护
日志审计：全量对话日志、独立成本日志、支持筛选回溯与报表导出
弹性扩容：模块化架构+预留席位，标准化流程快速新增Agent，资源完全隔离
冲突防护：文件统一管理、发言格式规范、分组权限管控
2. 整体架构（三层模块化架构，专为扩容设计）
采用核心中枢层 + 常驻执行层 + 弹性扩展层分层设计，层与层之间资源隔离，新增Agent仅在扩展层操作，不改动原有核心与常驻配置。
核心中枢层（不可动）
载体：OpenClaw
职责：群聊路由、进程托管、消息广播、任务调度、全局Token/成本统计、预算管控、集群权限管理；是整个集群唯一入口，扩容时无需修改。
常驻执行层（固定岗位，基础团队）
载体：MiMo Code、CodeWhale、CodexCLI、OpenCode
定位：核心固定员工，承担常规开发全流程，永久在线，配置固化。
弹性扩展层（预留扩容区，重点）
定位：专门用于后续新增各类Agent，提前预留席位、目录、日志、预算分组、广播分组；每一个新Agent独立进程、独立计量、独立日志，互不干扰。
3. 通信与资源规则
通信：基于OpenClaw ACP协议，全局双向广播，支持全局群聊 / 分组群聊双模式（扩容后防刷屏）
资源隔离：扩展层Agent拥有独立日志、独立成本台账、独立预算配额、独立工作子目录
统计规则：所有Agent（常驻+新增）单独计量Token、单独核算成本，全局汇总总用量

五、核心章节：弹性扩容 - 新增Agent标准化流程（重点）
当需要接入新Agent（新编码工具、专项AI、第三方Agent等）时，严格按照以下通用流程操作，全程不改动原有常驻配置，复用预留席位与目录。
1. 扩容前置规则（必遵守）
优先使用预留席位ID（ext01~ext06），不新增无规则名称；
新Agent工作目录绑定扩展席位/对应席位文件夹；
必须开启--track-usage true，独立统计Token与成本；
按业务归属加入预设分组（扩展组A/B/C）；
单独配置预算、单独生成成本日志，隔离数据。
2. 通用接入模板（复制即用，替换参数即可）
模板参数说明
[席位ID]：ext01 / ext02 ... ext06
[Agent执行命令]：新Agent终端启动命令（如xxxagent）
[分组名称]：扩展组A / 扩展组B / 扩展组C
第一步：挂载新Agent（OpenClaw内执行）

# 标准挂载命令（替换[]内内容）
/acp spawn --name [席位ID] --cmd [Agent执行命令] --track-usage true --group [分组名称] --workdir "./扩展席位/[席位ID]"
# 示例（接入第一个新Agent，占用ext01，归属扩展组A）
/acp spawn --name ext01 --cmd newagent --track-usage true --group 扩展组A --workdir "./扩展席位/ext01"
第二步：为新Agent配置独立预算

# 格式：/usage set-budget agent [席位ID] 单日预算金额
/usage set-budget agent ext01 20
第三步：配置分组广播（按需选择）

# 方式1：加入全局群聊（和所有常驻+扩展Agent互通，全员可见）
/acp group broadcast [分组名称] global on
# 方式2：仅分组内聊天（只和同组扩展Agent互通，防刷屏，推荐多扩容后使用）
/acp group broadcast [分组名称] inner on
第四步：新增独立成本日志（单Agent台账）

# 为该扩展席位单独导出成本日志，按席位归档
/usage export csv --agent [席位ID] --path "./全局日志/成本报表/[席位ID]_成本记录.csv" --interval 5
第五步：校验接入成功

# 查看所有成员（常驻+新增扩展Agent）
/acp list
# 查看新Agent用量追踪、分组、工作目录状态
/acp status [席位ID] --verbose
# 查看预算状态
/usage budget status agent [席位ID]
3. 多Agent批量扩容操作
若一次性接入多个新Agent，循环执行上述模板即可，示例：

# 批量挂载 ext01 ext02
/acp spawn --name ext01 --cmd agent1 --track-usage true --group 扩展组A --workdir "./扩展席位/ext01"
/acp spawn --name ext02 --cmd agent2 --track-usage true --group 扩展组A --workdir "./扩展席位/ext02"

# 批量开启分组广播
/acp group broadcast 扩展组A inner on
4. 席位释放（Agent下线/淘汰）
若某个扩展Agent不再使用，释放席位，后续可重复利用：

# 1. 终止进程
/acp kill [席位ID]
# 2. 清空该席位预算
/usage clear-budget agent [席位ID]
# 3. 关闭对应日志导出

八、风险防控方案（含扩容后常见问题）
1. 文件冲突（常驻+多扩展Agent并发修改）
规则：所有扩展Agent仅在自身独立目录工作，公共文件必须提交至中枢统一合并；
禁止跨目录直接修改其他Agent文件。
2. 消息刷屏（扩容后Agent数量多）
解决方案：启用分组内广播，非必要不开启全局互通；
严格执行【XX-工作】/【XX-对话】标签，区分内容。
3. Token/成本统计错乱
要求：所有Agent必须开启--track-usage true，不手动关闭追踪；
定期导出CSV报表，核对数据，异常时重启对应Agent。
4. 进程过载（接入大量Agent）
限制：单集群同时在线Agent建议≤10个；超出则拆分多个OpenClaw集群；
闲置扩展Agent及时kill释放进程。
5. 席位重名错误
强制使用预设ext01~ext06命名，禁止自定义随机名称。
