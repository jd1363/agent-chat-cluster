## 2026-06-20 14:03
- 主人提出 Agent Chat 从 MVP 升级到真正系统的关键判断，已写入 `PROJECT_PLAN.md` 的“从 MVP 到系统级架构升级”章节。
- 关键判断：项目已是“可运行的受控多智能体系统原型”，但仍处于“工程化 MVP 完成 → 系统化架构缺失”的临界点；能跑，但还不是完整系统。
- 当前核心缺口：
  1. 缺少真正 Scheduler：优先级调度、资源分配、并发控制、backpressure、自动选 agent、自动重试、负载均衡。
  2. 缺少 Event Bus/Event Layer：应从 command→script→result 升级为 event→queue→handler→state update，支持 event store、event replay、async dispatch。
  3. Agent 仍非自治：当前更像 worker，不自主决策/规划/选工具。
  4. 缺少统一 State Store：需要统一 task/agent/cost/message/scheduler state，形成 source of truth。
  5. 缺少系统级容错模型：需要 distributed retry policy、idempotency guarantee、recovery orchestration、dead letter queue。
- 下一关键路线：先抽象 Event Layer，再引入 Scheduler，最后统一 State Store。
- 架构图任务：把现有系统重构成一张可论文/可工程实现的系统级架构图，包含 Event Bus + Scheduler + Unified State Store + Agent Workers + Audit/Policy Plane。
