## 2026-06-25

### 真实执行引擎接入 ✅

**目标**：把 7 个 Agent 从"占位席位"变成真实可执行的 CLI 工具。

#### 完成的工作

1. **config/agents.json 更新** — 7 个 enabled Agent 全部加了 `displayName` + `executor` 字段：
   | Agent ID | displayName | CLI 命令 |
   |---|---|---|
   | agent-exec-01 | Codex | `codex exec "{prompt}"` |
   | agent-ext-01 | CodeWhale | `codewhale exec "{prompt}"` |
   | agent-ext-02 | OpenCode | `opencode run "{prompt}"` |
   | agent-ext-03 | Ollama | `ollama run maayan/Qwen2.5-14B-Instruct-GGUF:latest "{prompt}"` |
   | agent-ext-04 | MiMo | `mimo run "{prompt}"` |
   | agent-ext-05 | Codex-2 | `codex exec "{prompt}"` |
   | agent-ext-06 | CodeWhale-2 | `codewhale exec "{prompt}"` |

2. **scripts/executor_bridge.py 编写** — 桥接器核心功能：
   - 读取 Agent executor 配置 → 替换 {prompt} 占位符 → subprocess.run(shell=True) 调用真实 CLI
   - 捕获 stdout/stderr，超时处理，输出截断
   - 写结果文件到 tasks/dispatch/{task_id}-result.txt
   - 加排他锁更新 tasks.json 任务状态
   - 写审计日志
   - 支持 --dry-run 和 --timeout

3. **端到端测试** — 两条链路验证通过：
   - **Ollama (agent-ext-03)**: Task-042，58.6 秒完成，输出正常
   - **Codex (agent-exec-01)**: Task-044，185.9 秒完成，Codex 实际执行了 echo hello 并写回结果
   - openclaw_executor --collect 也能正确收集 executor_bridge 的结果

4. **Web Dashboard 更新** — server.py 的 /api/agents 返回 displayName/executorCommand/executorType，dashboard.html 显示 Agent 真实名字和引擎标签

#### 发现的问题
- Codex CLI 启动较慢（~3 分钟），需要较长超时（建议 300s+）
- Ollama 本地模型指令遵循能力有限（14B 模型会"创造"任务而不是执行任务），但链路本身正常
- executor_bridge 的 subprocess.run timeout 参数工作正常，能正确捕获 TimeoutExpired

### 系统集成三连改进 ✅ (10:40-10:50)

#### 1. executor_bridge 进程组 kill 优化 ✅
- 改动：`subprocess.run` → `subprocess.Popen` + 进程组管理

- 超时流程：`terminate()` → 等 3 秒 → kill 进程组 → `wait()` 回收
- 语法验证通过，dry-run 正常
- git commit: `7d91db4`

- 新增 `--execute-real` CLI 参数

- dry-run 兼容
- git commit: `7d91db4`（和上面一起提交）
