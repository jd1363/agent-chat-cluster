### 系统集成三连改进 ✅ (10:30-11:15)

#### 1. executor_bridge 进程组 kill 优化 ✅
- 改动：`subprocess.run` → `subprocess.Popen` + 进程组管理
- Windows: `CREATE_NEW_PROCESS_GROUP` + `taskkill /T /F` 递归杀进程树
- Unix: `os.setsid` + `os.killpg(SIGKILL)`
- 超时流程：`terminate()` → 等 3 秒 → kill 进程组 → `wait()` 回收
- `import signal` 已添加
- 语法验证通过，dry-run 正常
- git commit: `7d91db4`

#### 2. dispatch_task.py --execute-real 集成 ✅
- 新增 `--execute-real` CLI 参数
- 逻辑：检查 executor_bridge.py 存在 → 检查 prompt 文件存在 → 调用 executor_bridge
- 支持 `--execute --execute-real` 组合（先生成 prompt，再执行 CLI）
- dry-run 兼容
- git commit: `7d91db4`（和上面一起提交）

#### 3. Web Dashboard 执行按钮 ✅
- server.py: POST `/api/tasks/execute` 端点，用 `subprocess.Popen` 异步执行 dispatch_task.py
- dashboard.html: pending 任务显示 ▶ Execute 按钮，点击后 confirm → POST → 按钮状态变化 → 2 秒后刷新
- 前端通过 SSE 自动看任务状态变化
- git commit: `52f15b8`
