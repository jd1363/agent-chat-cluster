# 2026-06-25 天气项目升级 + CLI Agent 修复

## 核心突破：CLI Agent 非交互模式修复

### 问题
executor_bridge 调用 CLI Agent（codewhale/mimo/opencode）做开发任务时，Agent 完全不理解任务，输出"请补充需求"。

### 根因
1. **没有加自动批准参数** — CLI 非交互模式默认只读不写
   - CodeWhale: `exec` 不加 `--auto` 就只是纯文本聊天
   - MiMo/OpenCode: `run` 不加 `--dangerously-skip-permissions` 就不会写文件
2. **上下文注入反而帮倒忙** — 5000 字符的项目上下文塞进 prompt，导致命令行参数过长被截断
3. **工作目录没指向目标项目** — CLI 在 agent-chat-cluster 目录下运行，不是天气项目目录

### 修复
1. agents.json 配置更新：
   - CodeWhale args: `["exec", "--auto", "{prompt}"]`
   - MiMo args: `["run", "--dangerously-skip-permissions", "{prompt}"]`
   - OpenCode args: `["run", "--dangerously-skip-permissions", "{prompt}"]`
2. executor_bridge.py 改用 `shell=False` + args 列表模式（避免 shell 解析截断）
3. `--project` 模式下 workDir 改为目标项目目录
4. 去掉手动上下文注入（CLI 工具在 cwd 下自己扫描文件）
