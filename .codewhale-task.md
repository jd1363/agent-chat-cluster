## 功能要求

### 1. 存储位置

事件日志：

```text
logs/events/YYYY-MM-DD.jsonl
```

死信日志预留：

```text
logs/dead_letter/YYYY-MM-DD.jsonl
```

本次至少创建 `logs/events/`，`logs/dead_letter/` 可由脚本在需要时创建。

### 2. 事件结构

每条事件 JSON 至少包含：

```json
{
  "eventId": "EVT-YYYYMMDD-000001",
  "eventType": "task.created",
  "timestamp": "2026-06-20T06:10:00Z",
  "source": "control-plane",
  "correlationId": "Task-001",
  "causationId": null,
  "payload": {},
  "policySnapshot": {},
  "status": "pending"
}
```

字段要求：
- `eventId`：当天递增，格式 `EVT-YYYYMMDD-000001`。
- `timestamp`：UTC ISO 8601，结尾 `Z`。
- `eventType`：必填字符串。
- `source`：默认 `control-plane`。
- `correlationId`：可选。
- `causationId`：可选。
- `payload`：JSON object，默认 `{}`。
- `policySnapshot`：JSON object，默认 `{}`。
- `status`：默认 `pending`。

### 3. 并发安全

必须避免并发写入撞 eventId。

要求：
- 使用标准库实现跨进程锁。
- 可参考项目已有 `send_message.py` 的 `.state.lock` 思路。
- 状态文件可放：`logs/events/.state`。
- 锁文件可放：`logs/events/.state.lock`。

### 4. CLI

支持以下命令：

#### append

```bash
python scripts/event_log.py append --event-type task.created --source control-plane --correlation-id Task-001 --payload '{"title":"demo"}'
```

输出：
- 默认人类可读，包含 eventId 和写入路径。
- 支持 `--json`，输出完整事件 JSON。

#### list

```bash
python scripts/event_log.py list
python scripts/event_log.py list --date 2026-06-20
python scripts/event_log.py list --event-type task.created
python scripts/event_log.py list --correlation-id Task-001
python scripts/event_log.py list --limit 10
python scripts/event_log.py list --json
```

要求：
- 默认列出当天事件。
- 支持 date/event-type/correlation-id/limit 过滤。
- `--json` 输出 JSON array。

#### replay dry-run

```bash
python scripts/event_log.py replay --date 2026-06-20 --dry-run
```

要求：
- 本阶段只支持 `--dry-run`。
- 输出将按顺序 replay 的事件数量和简要列表。
- 如果未提供 `--dry-run`，应拒绝执行并提示当前只支持 dry-run。
